Ride Share

Languages/Technologies: Ruby

The goal of this project was create an object oriented implementation of a ride share system. The learning outcomes of this project include:

  • Creating and instantiating classes with attributes
  • Creating class methods and instance methods within classes
  • Writing pseudocode and creating tests to drive the creation of the code
  • Use CSV files to pull data and create objects based on that data
  • Design classes and methods to handle single responsibility

This project also allowed me to explore relationships within classes. For example, the Trip dispatcher class has "trips", all trip objects have a "driver" and "passenger". Each passenger and driver also have the potential to have "trips".

Throughout the project, I implemented TDD and added several edge cases. An example of an edge case in this project is "returns zero if there are no trips" within driver_spec total_revenue method. It's an edge case because there might be instances in which a driver has no trips. If a driver has no trips then there will be no trip costs (nil) and that can break the calculation. This test is assuring that this case is considered in the design.
